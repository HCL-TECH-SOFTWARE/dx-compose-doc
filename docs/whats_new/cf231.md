# What's new in CF231 

The following features and updates are available to customers installing HCL Digital Experience (DX) Compose Container Update CF231 on supported platforms:

- DAM - External database support now available
- DX Picker integration with Search V2
- Script for Liberty Status table management now available
- Search V2 - New Atomic Components and supported CSS part attributes available
- Site Builder is now available in HCL DX Compose
- OpenLiberty and JRE versions

## DAM - External database support now available

You can now configure external databases for Digital Asset Management (DAM). For more information and a sample scenario, refer to [Configuring an external database for DAM](https://help.hcl-software.com/digital-experience/9.5/latest/manage_content/digital_assets/configuration/external_dam_database/){target="_blank"}.

## DX Picker integration with Search V2

The DX Picker feature is now integrated in Search V2. This integration lets you search and select items from multiple content sources, including Java Content Repository (JCR) and Web Content Manager (WCM), in addition to Digital Asset Management (DAM). Installation and deployment methods have also been streamlined for efficiency. For more information, refer to the following topics:

- [Enabling and disabling DX Picker](https://help.hcl-software.com/digital-experience/9.5/latest/manage_content/wcm_authoring/dx_picker/enable/){target="_blank"}
- [Accessing DX Picker](https://help.hcl-software.com/digital-experience/9.5/latest/manage_content/wcm_authoring/dx_picker/access/){target="_blank"}
- [Using DX Picker](https://help.hcl-software.com/digital-experience/9.5/latest/manage_content/wcm_authoring/dx_picker/usage/){target="_blank"}
- [Limitations of DX Picker](https://help.hcl-software.com/digital-experience/9.5/latest/manage_content/wcm_authoring/dx_picker/limitations/){target="_blank"}

## Script for Liberty Status table management now available

You can now manage custom entries (create, read, update, delete) in the `LIBERTY_STATUS` table using the `custom_liberty_status.sh` helper function. Use this script to set, track, and control key-value pairs across your custom deployment's startup and update scripts. For more information on how to include and call this script from your custom startup or update scripts, refer to [Managing the Liberty Status table in custom scripts](../deploy_dx/manage/cfg_webengine/custom_liberty_status.md).
